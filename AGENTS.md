# コーディングにおいて大事にすること
- 簡潔さ
    - 例外についてはearly contine, return, raiseを行う
    - 同じ文字列を複数回書くことはなるべく避ける
    - 不要な変数定義をしない、できる限り行数を減らす
    - 後方置換性のためなどに、不要になったコードを残さない
- 頑健性
    - 基本的にどんな言語であっても型安全性を重視する、常に適切なlinterを導入して型チェックの自動化パイプラインを作成する
- 凝縮性
    - 意味のある単位でまとめる、不用意にmethodや関数を分けないこと
    - 関数名を見て何を入力して、何が返るのかが明快な関数の組み合わせにするべき。そうでないのであれば意味のない入出力の型を作るのではなく、関数のまとめ方がおかしいので全体を考え直すこと
- 可読性
    - 初めて読む人やdomain knowledgeがない人でも理解できるコードにすること
    - 使う単語や品詞にはこだわりを持つこと
    - 不要なcomment outをしない
- アーキテクチャに対する美意識
    - DDDやオニオンアーキテクチャなど、なんらかの思想に基づいたディレクトリ構成を行う
    - それぞれのレイヤーではどのような処理を書いてよくて、どのような処理を書いてはダメなのかのガイドラインを明確にする
    - 各レイヤーはどのレイヤーがどのレイヤーに依存していいのかを明確にする
