# UI/デザイン

## 方針

- shadcn/uiのコンポーネントを基準に拡張する。
- デザイントークンはCSS変数/Tailwindテーマに集約し、インラインスタイルを避ける。
- 色/余白/タイポグラフィはトークン経由で適用し、直接値を埋め込まない。

## デザイントークン運用

- `:root` かテーマ定義にトークンを集約する。
- 直接値の使用は例外にし、理由がある場合のみ許可する。
- 新規トークン追加は命名規則に従いレビュー対象にする。

## shadcn/ui運用

- ベースコンポーネントは変更せず、必要ならラッパーで拡張する。
- `components/ui` からの直接インポートを許可するが、外部公開は `src/components` で統一する。

## レイアウト指針

- 余白・幅・高さはトークンまたはユーティリティクラスで統一する。
- ページ単位のレイアウトは `modules/<domain>/pages` に閉じる。
