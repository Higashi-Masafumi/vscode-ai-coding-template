# バリデーション

## 方針

- フォーム/入力はzodスキーマを基準にし、UIからスキーマを参照する。
- 例外処理は早期returnで簡潔にする。

## スキーマ配置

- 汎用スキーマは `src/lib` ではなく `modules/<domain>/models` へ置く。
- 複数ドメインで共有する場合のみ `src/lib/validation` を許可する。

## UIとの接続

- フォーム側はスキーマから型を推論し、二重定義を避ける。
- エラーメッセージはスキーマに寄せ、UIは表示のみ行う。

## サーバー由来のエラー

- APIエラーはスキーマエラーと分け、フォームにマッピングして返す。
- フォームの状態管理は単一の責務に寄せる。
